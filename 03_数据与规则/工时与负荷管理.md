## 一、核心概念定义

### 1.1 工作负荷 (Workload)

工程设计人员的工时投入总和，包含以下类别：
- 项目工时
- 报价工时
- 年假工时
- 培训工时
- 业务建设工时

### 1.2 标准工时

- 基于**工作日**计算
- 受**标准工时折合比**影响
- 作为负荷率计算的分母基准

### 1.3 负荷率

**工程设计人员负荷率：**
```
负荷率 = (项目工时 + 报价工时 + 年假工时 + 培训工时 + 业务建设工时) ÷ 当期标准工时
```

**工程设计人员实际项目负荷率：**
```
实际项目负荷率 = 考核期内有合同收入的项目确认工时 ÷ 当期标准工时
```

**预测项目负荷率：**
```
预测项目负荷率 = 考核期内专业负责人预派发给工程设计人员的计划工时 ÷ 当期标准工时
```

## 二、工时计算规则

### 2.1 项目计算工时

计算工时根据项目签署的**去税合同额**与**综合基准费率**计算，用于指导项目报价和设定计划工时（预算工时）。

**一般设计咨询项目：**
```
计算工时 = (项目设计咨询部分合同金额 × 项目人工成本比例) ÷ 板块综合基准费率
```

### 2.2 工时成本计算

```
工时成本 = 工时 × 主专业资格费率
```

- 填报**本部门**项目工时：按人员的主专业资格对应等级查找费率
- 填报**其他板块**项目工时：使用 Workshare 费率
- **自有管理人员**的费率始终为 0

**示例：**
> 非自有管理人员甲，五个等级费率分别为 100、200、300、400、500，主专业资格为审核（四级）。填写 8 小时本部门工时 → 成本 = 8 × 400 = 3,200 元

### 2.3 怠工成本

```
怠工成本 = 怠工工时 × 主专业资格费率
```
- 也称 低负荷成本，每日计算，每月更新当年累计
- 除一下情况，无论是否参与工作负荷率考核，都会计算
	- 当此人为自有管理人员
	- 派遣人员不计算低负荷成本

## 三、Workshare 机制

### 3.1 Workshare 定义与判定规则

**什么是 Workshare？**
公司通过 Workshare 项目分工合作，推动不同运营单位间的协作，使高人力成本单位最大限度地使用低成本执行中心（HVEC），降低整体成本。

**判定规则：**

| 场景 | 是否算 Workshare | 费率依据 |
| :-- | :-- | :-- |
| 专业室工程师 → 给各板块做项目 | ❌ **不算** | 本部门人员费率 |
| 板块工程师 → 做**本板块**项目 | ❌ **不算** | 本部门人员费率 |
| 板块工程师 → 做**其他板块**项目 | ✅ **算** | **Workshare 费率** |

**核心原则：** 只有跨板块的工时才计入 Workshare，专业室与板块之间的协作属于正常工作分工。

### 3.2 HVEC 高价值低成本执行中心

| HVEC 中心 | 定位 |
| :-- | :-- |
| **银川中心** | 高价值低成本执行中心 |
| **沈阳中心** | 高价值低成本执行中心 |

- HVEC 对板块的 Workshare 费率**更低**，是公司的成本优势资源
- 公司鼓励高成本板块将工作分包给 HVEC，实现整体降本

### 3.3 Workshare 的战略价值

1. **降低成本**：高成本单位最大限度使用 HVEC，降低项目执行整体成本
2. **增强竞争力**：成本优势转化为市场竞争力
3. **灵活调配**：全公司范围内人力资源调配更加灵活
4. **提高负荷**：提升工程师整体人力利用率
5. **优化结构**：优化人员结构，避免人员闲置，减轻局部人力不足

### 3.4 Workshare 费率计算

- Workshare 工时使用**单独的 Workshare 费率**（非本部门人员费率）
- 部门费率（Workshare 费率表）参见 **CHN-HRM-PRO-900001**
- HVEC（银川/沈阳）对板块的 Workshare 费率 < 其他板块间的 Workshare 费率

## 四、费率体系

### 4.1 资格等级

| 序号 | 资格等级 | 对应资格 |
| :-- | :-- | :-- |
| 1 | 一级 | 制图 |
| 2 | 二级 | 设计、初级、初审、专员、项目工程控制服务、综合资格 |
| 3 | 三级 | 校核、中级、复审、专业经理、主管 |
| 4 | 四级 | 审核、高级 |
| 5 | 五级 | 审定 |

### 4.2 人员费率规则

- **一人一费率**：每个人有单独的五级费率
- 以**板块的各级平均费率**进行初始化
- 费率模式：目前按**主专业资格**计算工程师上报工时的费率
- 部门费率（Workshare 费率）参见 CHN-HRM-PRO-900001

### 4.3 综合基准费率

综合基准费率根据板块/中心/分公司所在地的设计人员的**项目执行全成本**计算所得。

**全成本包含：**
- 工程设计人员基本工资
- 五险一金
- 办公成本
- IT 成本
- 板块/分公司管理人员成本分摊
- 项目辅助人员（ES 等）成本分摊

**不包含：**
- 公司管理费
- 股东利润
- 税

### 4.4 工时成本占比

```
工时成本占比 = (预计总工时数 × 固定费率) ÷ 预估设计费或合同额 × 100%
```

**颜色提示逻辑：**

| 工时成本占比 | 背景色 |
| :-- | :-- |
| < 20% | 🔴 红色 |
| 20% ~ 50% | 🟢 绿色 |
| > 50% | 🔴 红色 |

| 单位 | 成本 Rate | 工资+奖金+社保 |

## 五、低负荷成本规则

- 无论是否参与工作负荷率计算
- **自有管理人员**：不管什么状态，**不计算**低负荷成本
- **派遣人员**：**不计算**低负荷成本

## 六、相关流程与场景

### 6.1 工时上报流程

1. 工程师按日填报工时（项目工时、非项目工时等）
2. 专业负责人审核确认
3. 系统按人员费率自动计算工时成本
4. 月末汇总生成工时成本占比等经营指标

### 6.2 负荷率应用场景

- **产能规划**：通过预测项目负荷率评估未来产能
- **人员调配**：识别低负荷/高负荷人员，优化资源分配
- **绩效考核**：实际项目负荷率作为工程师考核指标之一
- **成本控制**：低负荷成本识别产能闲置损失

### 6.3 Workshare 应用场景

- **项目分包决策**：高成本板块将非核心工作分包给 HVEC（银川/沈阳）
- **成本控制**：通过 Workshare 费率差实现项目整体降本
- **产能平衡**：低负荷中心承接高负荷中心的溢出工作量
- **跨区协作**：打破地理与组织边界，实现全公司人力最优配置

### 6.4 月度经营数据关联

- 工时数据与月度经营数据（存量合同额、WIP、应收账款等）联动
- 工时成本占比是项目盈利能力的关键指标
- 负荷率数据支撑业务预测与产能评估
- Workshare 工时占比是衡量跨区协作程度的重要指标

## 七、注意事项

- ⚠️ 工时填报需准确区分**本部门项目**与**跨板块项目**（费率不同）
- ⚠️ 专业室→板块的工时**不算** Workshare，按本部门费率计算
- ⚠️ 板块→其他板块的工时**算** Workshare，按 Workshare 费率计算
- ⚠️ HVEC（银川/沈阳）的 Workshare 费率低于其他板块，优先使用
- ⚠️ 存量类指标为累计值，需确保与上月数据衔接
- ⚠️ 流量类指标为当月值，仅统计本月发生额
- ⚠️ 同一板块不同目标类型的数据不可合并计算
