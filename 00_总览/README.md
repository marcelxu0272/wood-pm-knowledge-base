> Wood China 业务平台产品团队知识库
> 最后更新: 2026-04-24

## 📖 知识库概述

本知识库服务**504 业务平台**产品团队，沉淀平台功能、流程、规则、术语、组织的统一口径，减少沟通歧义与返工。

源数据来自 Notion 导出的「📚 504 平台字典」数据库，经整理后按产品经理的使用场景重新组织。

## 🗂️ 目录结构

| 目录 | 内容 | 使用场景 |
|------|------|----------|
| `00_总览` | 知识库入口、导航图 | 快速了解知识库全貌 |
| `01_平台架构` | 平台模块、功能清单、子系统 | 写 PRD 时查阅"平台有什么能力" |
| `02_运营流程` | 业务流程、状态流转、角色协作 | 画流程图、理解"事情怎么走" |
| `03_数据与规则` | 数据口径、计算规则、设计规范 | 写看板 PRD 时对齐"怎么算/怎么展示" |
| `04_概念术语` | 术语定义、概念边界、易混点 | 消除歧义，统一语言 |
| `05_组织与业务` | 组织架构、业务板块、项目类型 | 理解"谁负责什么"和业务背景 |
| `06_设计资产` | 主题色、IP 形象、视觉规范 | 设计 UI/UX 时参考 |

## 🧭 检索指南

遇到问题时按以下顺序查找：

1. **"平台有什么功能？"** → `01_平台架构/`
2. **"这个流程怎么走？"** → `02_运营流程/`
3. **"这个数怎么算/怎么展示？"** → `03_数据与规则/`
4. **"这个词是什么意思？"** → `04_概念术语/`
5. **"这个部门/角色负责什么？"** → `05_组织与业务/`
6. **"设计用啥颜色/用什么IP？"** → `06_设计资产/`

## 📐 知识库分类体系（来自 Notion 原始结构）

原始知识库按 6 大一级分类组织：

| 分类 | 定义 | 回答的问题 |
|------|------|------------|
| **Index** 索引 | 导航与入口 | "我想找入口/地图/阅读路径" |
| **Platform** 平台模块 | 功能、页面、交互、权限 | "平台有什么能力/怎么用" |
| **Operation** 运营流程 | 流程、步骤、状态流转 | "事情怎么流转/下一步谁做" |
| **Rules** 规则口径 | 计算规则、命名规范、验收标准 | "必须怎么填/怎么算/怎么验收" |
| **Concept** 名词概念 | 术语定义、概念边界 | "这个词到底是什么意思" |
| **Organization** 组织架构 | 部门、角色、职责边界 | "谁负责什么/组织怎么分" |
