# 工时上报明细数据结构

| 标签 | Data Structure |
| --- | --- |
| 数据更新时间 | 2026/05/21 |
| 数据源 | 504平台工时上报导出 |

## 概述

工时上报明细是平台记录工程师每日工时投入的核心数据表，每条记录代表**一名工程师在某一项目上的单日工时上报**。

- **总字段数**：23 个
- **文件命名规则**：`工时上报明细统计_{项目编码}.xlsx`
- **示例文件**：`工时上报明细统计_B25340.xlsx`（29条记录，1个项目，6名工程师）

## 数据计算逻辑

### 工时成本计算

```
已审工时成本 = 已审工时(H) × 费率
```

### 费率体系

费率由平台根据工程师的**管理归属部门**和**资格等级**自动匹配。示例文件中的费率分布：

- **¥94.0/H**（4条记录）
- **¥160.0/H**（2条记录）
- **¥185.1/H**（14条记录）
- **¥192.6/H**（1条记录）
- **¥196.1/H**（1条记录）
- **¥215.1/H**（3条记录）
- **¥220.5/H**（1条记录）
- **¥315.6/H**（3条记录）

### 工时状态流转

| 状态 | 字段 | 说明 |
| --- | --- | --- |
| 提交 | 提交工时(H) | 工程师填报并提交 |
| 待审 | 待审工时(H) | 等待专业负责人/PM审批 |
| 已审 | 已审工时(H) | 审批通过，进入成本核算 |

## 数据实体关系

```
项目 (项目编码, 项目名称, 板块归属)
  │
  ├── 单元 (单元号, 单元名称)
  │   │
  │   └── 工时记录 (日期, 工程师, 工时, 成本)
  │
  ├── 项目经理 (姓名, 管理归属)
  └── 专业负责人 (姓名, 管理归属)

工程师 (姓名, 管理归属)
  ├── 主专业 / 主专业资格
  └── 专业 / 资格 (项目维度)
  └── 费率 (部门 × 资格)
```

## 典型使用场景

| 场景 | 说明 |
| --- | --- |
| **项目成本核算** | 汇总某项目编码下所有已审工时及成本 |
| **部门工时统计** | 按工程师管理归属聚合工时，评估部门负荷 |
| **Workshare 分析** | 比对"工程师管理归属"与"板块归属"，识别跨板块协作 |
| **专业人力分析** | 按主专业标签聚合，评估各专业人力投入 |
| **费率核查** | 对比实际费率与基准费率，评估成本差异 |
| **趋势监测** | 按日期聚合，分析工时投入的时间分布 |

## 人员定义

### 工程设计人员

泛指从事公司设计、咨询、造价项目的各级设计岗位人员，并包括了文控、计划、文秘等项目服务人员和设计负责人。

### 自有管理人员

该类人员的人力成本已经分摊到其他人员的费率中，所以需要被排除在计算人力成本的人员范围之外。自有管理人员的费率始终为 0。
