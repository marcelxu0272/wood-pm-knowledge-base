## 三、基本原则

- 专业室是内部仅承担**成本/费用管控**、**不直接产生营业收入**的责任单位
- 不考核利润，重点考核：**成本、费用控制** 及 **设计交付质量 / 效率**
- **总原则**：专业室工程师（含室主任）参与项目获得的**项目工时**和**报价工时**与工程师对应的**岗位费率**，构成专业室的**内部收入**；当内部收入 > 实际成本时，可分配奖金

### 3.1 奖金分配范围

| 对象          | 分配人                      | 备案要求     |
| :---------- | :----------------------- | :------- |
| 专业室工程师      | 专业室主任（根据年度审批工时、个人收入贡献度等） | 报人力资源部备案 |
| 专业室主任（含副主任） | 由其主管领导考核和分配              | 报人力资源部备案 |
| 采购工程师团队     | 生态合作部总监及其主管领导            | 报人力资源部备案 |

### 3.2 关联因素

工程师的项目和报价工时、岗位费率、工程师个人内部收入等。

## 四、核算原则

### 4.1 符号定义

| 符号 | 含义 |
| :-- | :-- |
| N | 专业室年度内部收入总额 |
| C | 专业室年度实际发生总成本 |
| B | 年度可分配奖金（B = N − C） |
| N1 | 工程师个人内部收入 |
| C1 | 工程师个人实际成本 |
| R1 | 工程师费率 |
| Y% | 奖金系数 |
| MH | 已审工时 |

### 4.2 核算公式

**专业室层面：**
```
N = ∑ R1 × MH × (1 + Y%)    ← 专业室年度内部收入总额
B = N − C                     ← 年度可分配奖金
```

**工程师个人层面：**
```
N1 = ∑ R1 × 个人MH × (1 + Y%)   ← 工程师个人内部收入
```

### 4.3 说明

- **已审工时 MH**：包含年度已审核的**项目工时**和**报价工时**；业务建设、培训类工时已计入费率 R1 中考虑
- **专业室年度实际总成本 C**：年度所发生实际成本，含人工成本、IT、座位各种分摊成本、差旅成本等
- **工程师个人成本 C1**：年度个人实际发生的薪资类（含补贴）及五险一金企业等人工成本

## 五、兑现条件

### 5.1 B > 0（可分配奖金）

| 对象 | 规则 |
| :-- | :-- |
| 室主任（含副主任） | 由其主管领导**自行分配** |
| 工程师 | 由专业室主任进行分配 |
| 备案 | 上述分配报人力资源部备案；如有考核协议，提供考核协议为依据 |

### 5.2 B < 0（无可分配奖金额）

| 对象  | 规则                                        |
| :-- | :---------------------------------------- |
| 室主任 | **无奖金**                                   |
| 工程师 | 根据公司年度运营情况统一由公司评估是否给予奖金额度，如有额度则由专业室主任进行分配 |


### 5.3 个人资格判定（B < 0 时）

公司提供数据支撑，如工程师年度已审的项目工时、报价工时、个人内部收入等：

| 条件          | 结果                                   |
| :---------- | :----------------------------------- |
| N1 − C1 > 0 | **具备**在成本中心 B < 0 的情况下由公司统一分配奖金额度的资格 |
| N1 − C1 < 0 | **无资格**                              |

## 九、与平台 504 的关联

> 以下为本考核办法在 504 业务平台中的落地映射，供产品与运营团队参考。

### 9.1 数据需求

平台需要支持和展示以下关键数据指标：

| 指标 | 用途 | 计算方式 |
| :-- | :-- | :-- |
| 专业室内部收入 N | 考核成本中心业绩 | ∑ R1 × MH × (1 + Y%) |
| 专业室实际成本 C | 考核成本中心管控 | 人工成本 + IT + 座位分摊 + 差旅等 |
| 可分配奖金 B | 判定是否可分配 | B = N − C |
| 工程师个人收入 N1 | 判定个人奖金资格 | ∑ R1 × 个人MH × (1 + Y%) |
| 工程师个人成本 C1 | 判定个人奖金资格 | 薪资 + 补贴 + 五险一金企业部分 |
| 已审工时 MH | 核算基础 | 项目工时 + 报价工时 |

### 9.2 平台功能支撑

- **工时管理**：需支持取消 49 小时上限后的工时填报与审核
- **异常监测**：需具备工时异常填报/审批的识别与预警能力
- **成本核算**：需能按专业室维度归集成本 C（人工、IT、座位、差旅）
- **奖金看板**：需支持 N/C/B 指标的专业室维度及个人维度展示
- **否决项记录**：需与质量/安全/合规事件数据对接

## 十、2026-05-21 会议讨论要点

> 参会人员：李总（李永辉）、彭军（产品经理）、许月娟、徐亦盛（Ethan）、董总（董晓迪）

### 10.1 背景：为什么需要大范围调整

由于以下多项政策与机制同时变化，平台功能逻辑需要进行**大范围调整**：

| 变更 | 影响范围 | 说明 |
| :-- | :-- | :-- |
| **工时上报逻辑变更** | 工时填报、成本核算、看板展示 | 取消每周 49 小时上限，所有已审工时计入项目成本 |
| **取消绩效考核** | 负荷率计算、个人绩效看板 | 原《工程设计人员绩效考核办法》作废，考核体系从"绩效"转向"成本中心" |
| **取消 Workshare** | 跨板块协作、费率计算、降本分析 | Workshare 机制取消后，跨板块工时的费率计算逻辑需要重新定义 |
| **新考核办法上线** | 奖金核算、N/C/B 指标、个人资格判定 | 专业室作为成本中心，N > C 才可分配奖金 |

### 10.2 专业室 vs 个人奖金的独立核算

**核心发现：专业室和个人是两条线算。**

- 专业室整体没奖金（B < 0）≠ 个人没奖金
- 个人独立核算后（N1 − C1 > 0），即使部门 B < 0，个人仍可能获得公司统一分配的奖金
- 平台需同时支持**专业室维度**和**个人维度**的 N/C/B 指标展示

### 10.3 成本边界问题 — 业务建设

**待明确：什么是"业务建设"？**

- 专业室自己组织的业务建设 → **无额外成本**（不产生支出，不产生收入）
- 跨部门使用（如银川的人被专业室用来做业务建设）→ 成本归属不明确
- 需要判定逻辑：**"这个人和这个项目是否属于同一个部门"**，不是的就要算成本
- 吴新军发出去给各科室的任务 → 算不算各科室的收入？算不算部门的支出？**目前不清楚**

**李总决策：** 先简单化处理，6 月 1 日上线后再细化

### 10.4 新大学生可能成为"负资产"

- 新大学生对专业室来说可能是"负资产"（N1 < C1，挣不回自己的工资）
- **建议方案（待确认）：**
  - 给新大学生一定的"底薪"收入（虚拟收入计入 N1）
  - 或者核减新大学生的一部分成本（降低 C1）
  - 允许专业室招大学生时有一定的"保护期"折扣

### 10.5 人力资源看板 — 复核数据不打折

- **李总建议：** 复核直接显示客观真实数据，**不要给新来的人打折保护**
- **问题现状：** 好几百人的复核数据都是扁的（因为打折），看不出真实情况
- 取消 80% 界限后，建议数据"真实一些"
- 新大学生的保护方案可以单独考虑（见 10.4）

### 10.6 专业室/职能部门考核分工

| 考核对象 | 负责人 | 说明 |
| :-- | :-- | :-- |
| 专业室考核 | 李总 | 专业室作为成本中心的考核办法 |
| 板块考核 | 李总 | 各业务板块的考核 |
| 职能部门考核 | 老潘（潘建兴）牵头，董总协助 | 职能部门的考核体系 |

- 董总需本周发一份专业室考核方案给李总，以便下周管理会前交流

### 10.7 工时填报规则

- **填报界面维持现状**：工程师填工时的界面保持原样
- **不强制按周拆分**：即使核算按周，填报还是按天
- **员工自主决定频率**：系统提供每天填的工具，但员工可以每天填、最后补填，都可以
- **原则：** 工具提供便利，不强制约束
