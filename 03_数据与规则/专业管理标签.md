## 概述

平台中工程师的**专业管理标签**由两部分组成：

1. **专业**（Professional Specialty）：标识工程师所属的业务域、部门和专业方向，格式为 `{业务域}-{部门}-{专业}` 三级编码
2. **资格**（Qualification）：标识工程师的岗位资质等级或专业资格，用于工时成本核算、费率匹配和专业室看板管理

当前平台共维护 **108** 个专业标签、**17** 个资格等级标签，覆盖 **792** 名工程师。

## 标签使用场景

| 场景 | 使用方式 |
| --- | --- |
| **工时核算** | 专业标签用于确定工时成本占比和板块归属 |
| **专业室看板** | 按专业标签聚合展示各专业室人力资源与负荷 |
| **Workshare 分析** | 跨专业标签的工时记录用于评估板块间协作 |
| **费率匹配** | 资格标签 + 主专业匹配 config/rates.json 中的五级费率 |
| **岗位需求** | 按专业+资格维度统计岗位缺口和招聘需求 |
